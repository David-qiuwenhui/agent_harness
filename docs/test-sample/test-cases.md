## 五、使用方法

### 每次迭代后的自检流程

1. **启动环境** → 跑「第四部分：服务依赖验证」
2. **快速回归** → 跑「第一部分：基础能力」+ 「第二部分 2.1 + 2.2」
3. **详细验证** → 完整跑「第二部分 2.3：数据准确性」
4. **记录结果** → 在通过列打 ✅ 或 ❌，失败的记录具体表现

### 优先级

- **P0（阻断）**：2.1.1、2.2.1、2.2.3 — 核心功能和容错
- **P1（重要）**：2.1.3、2.1.4、2.3.x — 准确性和格式
- **P2（建议）**：第三部分 — 健壮性
